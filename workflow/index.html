Project workflow

To simplify the product development process, Penelope splits the project lifecycle into 5 distinct phases that provide the project with more predictability. The first phase consists of the Initiation Phase where the project is defined and a product vision is created. The second phase (Planning Phase) is the phase where all the goals and details are defined in order to complete what has been set on the first phase. The Execution Phase is characterised by the execution of the product, when all that has been defined comes to life. During the Control & Monitoring Phase, which happens alongside the previous phase, the Project manager is responsible for monitoring progress and making adjustments if necessary, ensuring that the product is aligned with what was defined in the project plan. At last, the Closure Phase dictates the end of the project and during this phase Penelope ensures that the product meets all the requirements defined in the first phase.

The workflow structure

The phases

Initiation Phase:
  • Define the project and create a product vision.
  • Capture customer requirements and ensure understanding.
  • Review and confirm client orders and requirements.
  • Establish effective customer communication channels.
Planning Phase:
  • Define project goals, roles, and responsibilities.
  • Create a project plan and establish logistics and communication channels.
  • Identify project risks and develop contingency plans.
  • Create a workflow/roadmap for the project.
Execution Phase:
  • Utilise version control to manage development branches.
  • Follow test-driven development to ensure quality and security.
  • Conduct unit testing, instrumentation testing, and user interface testing.
  • Utilise an automated test framework for user interface tests.
Control & Monitoring Phase:
  • Monitor project progress and address roadblocks.
  • Conduct status meetings and produce reports.
  • Manage project resources and allocate tasks.
  • Implement corrective and preventive actions.
Closing Phase:
  • Complete all required tests and ensure the product meets requirements.
  • Release the product to the customer or hand it over to another team.
  • Conduct a product review and obtain customer feedback.
  • Perform a project post-mortem to review project performance and identify improvements.
